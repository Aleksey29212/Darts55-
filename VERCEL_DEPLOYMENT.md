# 🚀 Руководство по развертыванию на Vercel

Vercel — это платформа от создателей Next.js, идеально подходящая для хостинга этого проекта. Развертывание происходит автоматически при каждом обновлении вашего GitHub-репозитория.

## Шаг 1: Подготовка
- Убедитесь, что ваш проект находится в репозитории на GitHub.
- Создайте аккаунт на [Vercel](https://vercel.com), если у вас его еще нет.

## Шаг 2: Создание проекта в Vercel
1.  Перейдите в ваш **Dashboard** на Vercel.
2.  Нажмите **Add New...** -> **Project**.
3.  В разделе **Import Git Repository** выберите ваш репозиторий с проектом DartBrig Pro. Vercel автоматически определит, что это Next.js проект.

## Шаг 3: Настройка переменных окружения
Это самый важный шаг. Vercel нужно предоставить все ключи и пароли, которые использует ваше приложение.

1.  В настройках созданного проекта перейдите во вкладку **Settings** -> **Environment Variables**.
2.  Добавьте следующие переменные:

    | Имя переменной                            | Значение                                       | Описание                                  |
    | ----------------------------------------- | ---------------------------------------------- | ----------------------------------------- |
    | `NEXT_PUBLIC_ADMIN_PASSWORD`              | `ваш_секретный_пароль`                          | Пароль для доступа в админ-панель         |
    | `GEMINI_API_KEY`                          | `ваш_ключ_от_google_ai`                        | Ключ для работы AI-функций (студия стилей) |
    | `NEXT_PUBLIC_FIREBASE_API_KEY`            | `...`                                          | Ключ API из вашего Firebase проекта        |
    | `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN`        | `...`                                          | Домен аутентификации Firebase             |
    | `NEXT_PUBLIC_FIREBASE_PROJECT_ID`         | `...`                                          | ID вашего Firebase проекта                |
    | `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET`     | `...`                                          | Корзина хранения Firebase                 |
    | `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID`| `...`                                          | ID отправителя сообщений Firebase         |
    | `NEXT_PUBLIC_FIREBASE_APP_ID`             | `...`                                          | ID вашего Firebase приложения             |

    **Опциональные переменные для системных обновлений:**
    
    | Имя переменной                            | Значение                                       | Описание                                  |
    | ----------------------------------------- | ---------------------------------------------- | ----------------------------------------- |
    | `GITHUB_TOKEN`                            | `ваш_токен_github`                              | Для запуска сборок из админ-панели        |
    | `GITHUB_OWNER`                            | `ваш_логин_github`                             | Владелец репозитория                      |
    | `GITHUB_REPO`                             | `название_репозитория`                         | Название репозитория на GitHub            |

    *Все переменные, начинающиеся с `NEXT_PUBLIC_`, будут доступны на стороне клиента. Остальные — только на сервере.*

## Шаг 4: Развертывание
1.  После добавления всех переменных перейдите во вкладку **Deployments**.
2.  Нажмите на кнопку **Deploy** для последней отправки в `main` ветку или просто сделайте новый `push` в репозиторий, чтобы запустить автоматическое развертывание.
3.  Vercel автоматически установит зависимости, соберет проект и развернет его.

## Шаг 5: Готово!
После завершения развертывания ваш сайт будет доступен по домену, предоставленному Vercel. Вы можете привязать свой собственный домен во вкладке **Settings** -> **Domains**.

---

### Преимущества Vercel для этого проекта:
- **Автоматические сборки:** Любое изменение в ветке `main` автоматически запускает новую сборку и развертывание.
- **Оптимизация:** Vercel автоматически применяет лучшие практики для производительности Next.js приложений, включая кеширование и CDN.
- **Простота:** Весь процесс сводится к настройке переменных окружения.
